# Курьер-раннер

Мобильная браузерная 2D endless runner игра в пиксельном стиле. Курьер на самокате едет по бесконечной дороге, перепрыгивает препятствия и собирает бонусы. В конце — начисление баллов и отправка на API.

**Ориентация:** только портретная. **Управление:** тап по экрану = прыжок.

## Структура проекта

```
courier-runner-game/
├── .cursorrules
├── index.html
├── css/style.css
├── js/
│   ├── config.js      # Константы, размеры, API, таблица наград
│   ├── game.js        # Главный цикл, экраны, связь модулей
│   ├── renderer.js    # Масштабирование canvas, DPR, resize
│   ├── input.js       # Тач-ввод, вибрация, запрет жестов
│   ├── player.js      # Курьер: физика, прыжок, анимация
│   ├── obstacles.js   # Препятствия (пул), генерация
│   ├── bonuses.js     # Бонусы (пул), лимит 50 за игру
│   ├── background.js  # Облака, земля, камешки
│   ├── collision.js   # AABB коллизии
│   ├── particles.js   # Частицы смерти и сбора (пул, макс 20)
│   ├── sound.js       # Web Audio: прыжок, сбор, удар
│   ├── api.js         # sendReward(), calculateReward()
│   └── ui.js          # Кнопки Game Over
├── manifest.json      # PWA (добавить на домашний экран)
└── README.md
```

## Запуск локально

Игра использует ES6-модули, поэтому нужен сервер (не открывать `index.html` как файл).

1. **Через Python:**
   ```bash
   cd courier-runner-game
   python -m http.server 8080
   ```
   Откройте в браузере: `http://localhost:8080`

2. **Через Node (npx):**
   ```bash
   npx serve .
   ```

3. **Через расширение Live Server в VS Code / Cursor**  
   Правый клик по `index.html` → «Open with Live Server».

## Тестирование на телефоне

1. Убедитесь, что ПК и телефон в одной Wi‑Fi сети.
2. Узнайте IP ПК (например, `ipconfig` в Windows, `ifconfig` на Mac/Linux).
3. Запустите локальный сервер (например, `python -m http.server 8080`).
4. На телефоне откройте в мобильном браузере: `http://192.168.x.x:8080` (подставьте свой IP).
5. Проверьте:
   - портретная ориентация;
   - тап в любом месте экрана = прыжок;
   - нет зума и pull-to-refresh;
   - вибрация при столкновении и сборе бонуса (если устройство поддерживает);
   - после Game Over кнопки «Забрать N ⭐» и «Ещё раз».

## Тестирование на ПК (эмуляция)

1. Chrome: F12 → Toggle Device Toolbar (Ctrl+Shift+M).
2. Выберите устройство, например iPhone 12 или Samsung Galaxy S20.
3. Ориентация: портрет.
4. Для прыжка можно использовать пробел или стрелку вверх (только для отладки).

## API и настройки

В `js/config.js` задаются:

- `API_URL` — базовый URL API (по умолчанию `https://api.example.com`).
- `API_KEY` — ключ для заголовка `Authorization: Bearer ...`.

Эндпоинт отправки награды: `POST {API_URL}/game/reward`.  
Параметр `user_id` берётся из URL: `?user_id=...`.

Таблица наград по метрам и формула (базовые баллы + бонусы × 2) описаны в `.cursorrules` и реализованы в `api.js` и `config.js`.

## Технологии

- HTML5, CSS3, JavaScript ES6+ (модули).
- Canvas 2D, виртуальное разрешение 400×600, масштабирование под экран и DPR.
- Без фреймворков и npm. Шрифт: Google Fonts «Press Start 2P».
